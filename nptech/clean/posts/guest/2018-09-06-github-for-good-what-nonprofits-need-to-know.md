To first understand how [**GitHub for Good**](https://github.com/nonprofit) works, let’s take a step back and understand what [**GitHub**](https://github.com/) actually is. At its simplest form, Github is a place that houses [**code**](https://www.economist.com/the-economist-explains/2015/09/08/what-is-code) into groups called [**repositories**](https://github.com/trending) and allows one or more people to update and keep [**version controlled**](https://en.wikipedia.org/wiki/Version_control) copies of their files.

**Imagine this scenario:** Joe and Alex are both web developers working on a nonprofit’s website who want to may sure they:

1. Do not overwrite each other’s work
2. Keep one central location for all their web files

The nonprofit has a Github Repository for *nonprofitname.org* and both developers add their files to the repository. Each time they add an update, both the current and older versions are saved. For the more technically advanced, you can build [**branches**](https://github.com/Kunena/Kunena-Forum/wiki/Create-a-new-branch-with-git-and-manage-branches) to create new functionality, do code reviews, get outside help, and merge fixes back into the original code. ## **Enter Github for Good**

In short, [**GitHub for Good**](https://github.com/nonprofit) is a free GitHub account. To qualify, your organization must meet a set of requirements:

**GitHub nonprofit accounts are designed for official nonprofit organizations and charities that are nongovernment, nonacademic, noncommercial, nonpolitical in nature, and have no religious affiliation.**

If your organization does not fall within their guidelines, you can [**find a plan**](https://github.com/pricing) that works for you.  Finally, it’s worth noting that GitHub is also compatible with many different project management systems, such as [**Basecamp**](https://basecamp.com/), [**Zapier**](https://zapier.com/), [**Slack**](https://slack.com), and [**Trello**](https://trello.com/).
